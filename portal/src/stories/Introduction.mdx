# WAVE Design System

Welcome to the WAVE Portal Design System documentation. This Storybook contains all design tokens, components, and patterns used throughout the application.

## Overview

WAVE uses a **Linear-inspired dark theme** with a focus on:
- **Monochromatic palette** (grays only for base colors)
- **High contrast** for accessibility (WCAG AA compliant)
- **Semantic color system** (success, warning, destructive)
- **Dark mode first** approach

## Architecture

### Technology Stack

- **UI Library**: Radix UI Primitives (headless components)
- **Styling**: Tailwind CSS with semantic CSS variables
- **Component Variants**: class-variance-authority (CVA)
- **Icons**: Lucide React (500+ icons)
- **Fonts**: Inter (UI) + JetBrains Mono (code)

### Design Tokens

All design tokens are defined as CSS variables in `src/index.css` and mapped through Tailwind:

```css
/* Dark Mode Example */
.dark {
  --background: 0 0% 12%;    /* #1e1e1e */
  --foreground: 0 0% 98%;    /* #fafafa */
  --border: 0 0% 18%;        /* #2e2e2e */
  --card: 0 0% 12%;          /* #1e1e1e */
  ...
}
```

## Design Principles

### 1. Dark-First Design
The canonical presentation is dark mode, optimized for extended coding sessions under any lighting condition.

### 2. Semantic Token System
Colors have meaning:
- **Success (green)**: Passed gates, completed tasks
- **Warning (amber)**: Pending states, attention needed
- **Destructive (red)**: Errors, failures, blocking issues

### 3. Monochromatic Base
The interface uses only grayscale colors for backgrounds, borders, and surfaces. Chromatic colors (green, amber, red) are reserved exclusively for semantic meaning.

### 4. Accessibility First
- All text meets WCAG AA contrast requirements (4.5:1 minimum)
- Focus indicators are clearly visible
- Color is never the sole indicator of state
