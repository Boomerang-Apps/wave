# /report - Progress Report Generation

**Priority:** P2 (MEDIUM)
**Recommended Model:** Haiku
**Aliases:** /progress, /summary

## Purpose

Generate progress reports for waves, stories, or overall project status. Outputs in Markdown, JSON, or Slack format for stakeholder updates.

## When to Run

- End of day status update
- Stakeholder check-ins
- Sprint reviews
- On-demand progress check

## Arguments

| Argument | Description | Default |
|----------|-------------|---------|
| `<wave>` | Wave number | Active wave |
| `--format` | Output format: md, json, slack | md |
| `--send` | Send to Slack channel | false |
| `--save` | Save to file | false |
| `--story <id>` | Report on specific story | - |

## Report Types

### Wave Progress Report
```bash
/report 1
```

### Story Report
```bash
/report --story WAVE1-FE-001
```

### Daily Summary
```bash
/report --format slack --send
```
